# T.I.A Public (Vitrine) — Knowledge Base para Agentes

> **Propósito deste documento:** Fornecer contexto técnico e arquitetural ao agente IA que irá trabalhar no novo repositório isolado (`tia-public`).

## 1. O que é o `tia-public`?

O `tia-public` é uma versão reduzida, segura e **100% anonimizada** (vitrine/portfólio) do projeto original RAG T.I.A.
Ele não contém dados sensíveis individuais (alunos, turmas, propriedades residenciais ou identificadores). Todo o banco de dados local que alimentará essa aplicação foi pre-agregado com inferências de Machine Learning operando a níveis de UF (Unidade Federativa) e Município.

### **Arquitetura**
* **Database**: Neo4j (populado estaticamente via arquivos `.cypher` na pasta `/neo4j/seed/`).
* **API**: FastAPI (Relying on Neo4j for Graph retrieval).
* **RAG**: Módulo customizado interligado a LLM para consultas naturais de perfis estatísticos dos estados.

## 2. O Schema do Grafo (Atenção Modificada)

O [schema original] completo contemplava estruturas como `Student`, `Classroom`, `School`, `StudentClass`, `Health`, etc.
**No `tia-public`, estas estruturas não existem!** O banco restringe-se exclusivamente aos agregados governamentais/demográficos.

### Nós Disponíveis no Grafo Público:

#### 🟢 `State` (Unidade Federativa)
É o nó raiz e contém todas as predições dos modelos (ML) em lote e métricas agregadas do IBGE/QEdu.
* **ID / Nome**: `sigla` (ex: "SP", "MG"), `name` (ex: "São Paulo")
* **Agregadores de Risco ML (Gerados por XGBoost e Scikit-Learn)**:
    * `pub_pct_risco_evasao_baixo` / `medio` / `alto` / `critico`: Distribuição % da propensão à evasão.
    * `pub_media_score_evasao`: Nota bruta do modelo global para o estado.
    * `pub_pct_reprovacao_predita`: Predição baseada em Random Forests de alunos sujeitos a nota < 5.
    * `pub_media_nota_predita`: Média das notas preditas.
* **Agregadores IBGE/Censo**:
    * Taxas de analfabetismo (ex: `branco_pnad_t_analf25m`), distorção idade-série, IDEB (`qedu_ideb_ai`), taxas de pobreza, infraestrutura racial urbana, e freqüência escolar do PNAD.

#### 🟢 `Municipality` (Município)
Cada Município está ligado ao Estado `(Municipality)-[:BELONGS_TO_STATE]->(State)`.
* **Propriedades**: Métricas do Atlas IBGE (`atl_freq_liq_fund`, atraso escolar, acessibilidade a laboratórios/PC formatados via recorte racial).

## 3. Direcionamentos CRÍTICOS para o Agente IA (Rules)

1. **Nunca busque alunos ou escolas diretamente**: Em qualquer request de prompt para cypher, saiba que nós como `Student` e `School` retornarão vazios. Tudo está consolidado nos nós `State` e `Municipality`.
2. **Escala de Avaliação de Risco**: As propriedades injetadas com prefixo `pub_` são as respostas do pipeline preditivo ML do data warehouse. `pub_pct_risco_evasao_critico` = % de perfis estruturalmente críticos identificados pelo XGBoost.
3. **Cypher Queries de Analytics**: Seu principal objetivo na criação de rotas ou novas lógicas de RAG é cruzar as métricas estaduais do ML (ex: `pub_pct_risco_evasao_critico`) com os dados demográficos (ex: `qedu_taxa_abandono` e `qedu_distorcao_ef_ai`) para que a LLM traga recomendações de políticas públicas ricas, sem envolver engenharia de dados.
4. **Tratamento IBGE Ausente**: Lembre-se da regra de ouro do T.I.A: Municípios de interior muitas vezes não possuem dados computados no IBGE. Use a função `coalesce` baseada na métrica macro do `State` correspondente quando não houver o valor municipal.
